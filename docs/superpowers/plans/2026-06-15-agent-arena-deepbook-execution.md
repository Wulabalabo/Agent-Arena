# Agent Arena DeepBook Execution Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace mock execution behind a feature flag with a DeepBook Predict Testnet execution adapter that reads Predict server state, confirms Sui objects, signs from platform-managed trading wallets, and records audit logs.

**Architecture:** Keep the platform API unchanged and introduce adapters under `src/platform/predict` and `src/platform/wallets`. Tests use mocked clients so the default test suite does not require network access or live Testnet funds. Live execution remains disabled unless explicit Testnet config and signer storage are present.

**Tech Stack:** Bun, TypeScript, `@mysten/sui`, DeepBook Predict public server, Sui Testnet, existing backend tests.

---

## File Structure

- Create `agent-arena/apps/backend/src/platform/predict/config.ts`: Testnet Predict configuration and feature flags.
- Create `agent-arena/apps/backend/src/platform/predict/server-client.ts`: typed client for Predict public server reads.
- Create `agent-arena/apps/backend/src/platform/predict/sui-client.ts`: confirmation-critical Sui read and submit adapter.
- Create `agent-arena/apps/backend/src/platform/wallets/key-store.ts`: encrypted or mock key store boundary.
- Create `agent-arena/apps/backend/src/platform/wallets/signer.ts`: signer facade that requires `intentId`, `riskDecisionId`, and `executionId`.
- Create `agent-arena/apps/backend/src/platform/wallets/signing-audit.ts`: audit record creation.
- Create `agent-arena/apps/backend/src/platform/executors/deepbook-executor.ts`: DeepBook Predict execution adapter.
- Modify `agent-arena/apps/backend/src/platform/execution.ts`: dispatch to mock or DeepBook executor by config.
- Add tests beside each new module.
